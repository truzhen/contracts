## 繁體中文

# 徒真（truzhen）Contracts

**信任不靠口頭承諾，靠白紙黑字的公開契約。**

想創作 Pack？請先看 [truzhen/packs](https://github.com/truzhen/packs)。本倉回答的是：這套市場為什麼做得成。

徒真（truzhen）讓開源作者和行業專家把能力做成 Pack，賣給中國中小企業。企業敢安裝陌生作者的能力，靠的不是相信作者人品，而是公開、穩定、機器可校驗的規則：Pack 能聲明什麼、哪些動作必須確認、做完留下什麼回執。

`github.com/truzhen/contracts` 就是這套規則的權威來源。它只包含 Go 類型和 JSON Schema，零實現、零副作用，不依賴任何實現倉。

### 三句話

1. AI 和 Pack 只能提建議，所有產出先是候選。
2. 發送、執行、寫正式資料等關鍵動作，必須經企業負責人確認和平台裁定。
3. 重要動作必須留下可回放的回執。

如果你只是做 Pack，通常不需要先讀完整 SDK；從 `truzhen/packs` 開始即可。當你要寫校驗器、工具鏈或改跨倉字段時，再回到本倉。

## English

# truzhen Contracts

**Trust is not built on verbal promises. It is built on public contracts.**

Want to create a Pack? Start with [truzhen/packs](https://github.com/truzhen/packs). This repository explains why the market can be trusted.

truzhen lets open-source maintainers and domain experts package their capabilities into Packs for small and mid-sized businesses in China. A business can install an unknown author's Pack only when the rules are public, stable, and machine-checkable: what a Pack may declare, which actions require confirmation, and what receipt remains afterward.

`github.com/truzhen/contracts` is the source of truth for those public rules. It contains Go types and JSON Schema only: no runtime implementation, no side effects, and no dependency on implementation repositories.

### Three rules

1. AI and Packs can only propose. Outputs start as candidates, not formal results.
2. Important actions such as sending, executing, or writing formal data require business-owner confirmation and platform adjudication.
3. Important actions must leave replayable receipts.

If you only want to submit a Pack, you usually do not need to read the full SDK first. Start from `truzhen/packs`. Use this repository when you build validators, tools, or cross-repository fields.
